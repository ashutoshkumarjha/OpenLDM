This section allows the inclusion of policy of land use by planner and enable cellular automata engine before model execution. It has following subsections.


    1. Policies

The policy inclusion includes defining 1) the retention capability of each of LULC class (i.e. class inertia) and 2) expected demand of LULC class in term of pixels. By default these are estimated from the transition probability matrix.

This section has provision of inclusion of retention capability of LULC classes in the range of 0-1.0.The value of 1.0 denotes high inertial/retention and zero denotes low inertia/retention. In other words if a value of 1.0 is assigned against LULC classes (under “Class” column heading) in row corresponding to class in “Class Inertia” column will result in class migration to be disabled. Similarly 0 in the fields means the classes can migrate to other new location. The values in between will result in few of the pixel to be in same grid while the reaming to be free to move anywhere in the lattice. In Figure 7 the class inertia is set to 1 for Built-up class.

The LULC modeling can also be modeled based on the future scenario of LULC class wise demand. Planner can enter the class wise future demand in number of pixels under “Demand” heading of LULC class against each LULC classes.

The above mentioned options will be enabled on the basis of enabling of these features by selecting “User Defined Demand” and “User Defined Class Inertia” checkbox.


    1. Spatial Context

This subsection has a provision to include neighboring distribution of particular class to be included in suitability map generation. To consider the weight “Window size” is set. This has iterative nature so the number of steps size of this iteration can be set by setting “Steps required” combo. The steps output generated in each iteration of time interval can also be generated if the user wants the output to be saved.


After setting above the two subsection output can be generated by executing the “Execute Button”
